    Unit Arithmetic API

    UDUNITS-2 Documentation Unit Arithmetic API

    Functions for constructing derived units through mathematical operations.

    All functions in this API return new unit instances rather than modifying existing ones.

    Functions

    Function Description
    ut_divide() Returns a new ut_unit representing the quotient of two units in the same ut_system
    ut_invert() Returns a new ut_unit representing the multiplicative inverse of a unit
    ut_log() Returns a logarithmic ut_unit defined relative to a reference unit in the same ut_system
    ut_multiply() Returns a new ut_unit representing the product of two units in the same ut_system
    ut_offset() Returns a new ut_unit defined by applying an additive offset to an existing unit
    ut_offset_by_time() Returns a new time-offset ut_unit relative to a temporal origin in the same ut_system
    ut_raise() Returns a new ut_unit raised to a specified power within the same ut_system
    ut_root() Returns a new ut_unit representing the specified root of a unit within the same ut_system
    ut_scale() Returns a new scaled ut_unit defined by applying a constant factor within the same ut_system

    1. Obtain ut_unit objects

      Operations begin with one or more existing ut_unit objects within a ut_system.

      ut_unit objects may originate from parsing a unit expression string or retrieving predefined units stored in the ut_system.

      Typical entry points include:

      • ut_parse()
      • ut_get_unit_by_name()
      • ut_get_unit_by_symbol()

      These functions produce structured ut_unit representations that can be combined using arithmetic operations.

    2. Construct derived ut_unit objects

      Arithmetic functions operate on compatible ut_unit objects belonging to the same ut_system to produce new derived units.

      • ut_multiply()
      • ut_divide()
      • ut_invert()
      • ut_raise()
      • ut_root()

      Each function returns a newly allocated ut_unit representing the resulting dimensional structure within the same ut_system.

    3. Intermediate dimensional representation

      The resulting ut_unit objects encode dimensional relationships independent of textual representation.

      Intermediate ut_unit objects may represent partially constructed derived units and can be reused in additional arithmetic operations within the same ut_system.

    4. Apply transformations to ut_unit

      Transformation functions modify magnitude or reference basis while preserving dimensional compatibility within the same ut_system.

      • ut_scale()
      • ut_offset()
      • ut_offset_by_time()
      • ut_log()

      These operations produce new ut_unit objects suitable for use in conversion or comparison workflows.

    5. Derived ut_unit available

      The resulting ut_unit represents a fully constructed derived unit defined within the same ut_system.

      This structured representation supports consistent use across parsing, conversion, formatting, and comparison workflows.

    6. Create cv_converter (optional)

      When numeric transformation between compatible units is required, ut_get_converter() produces a cv_converter describing the relationship between two compatible ut_unit objects within the same ut_system.

      The cv_converter enables transformation of numeric values while preserving dimensional consistency.

    7. Use derived ut_unit or cv_converter

      The resulting ut_unit or cv_converter can be used in application workflows requiring unit-aware computation or representation.

      Typical uses include:

      • numeric value transformation
      • formatting derived unit expressions
      • comparing dimensional compatibility
      • integrating derived units into application logic
