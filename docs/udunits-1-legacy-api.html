    UDUNITS-1 Legacy API

    UDUNITS-2 Documentation UDUNITS-1 Legacy API

    The UDUNITS-1 Legacy API provides backward-compatibility functions retained for applications originally written against the UDUNITS-1 interface.

    These functions internally adapt legacy workflows to the UDUNITS-2 object model, which is based on ut_system, ut_unit, and cv_converter objects.

    Because these functions predate the explicit ut_system object model, system management is handled internally.

    Note: New applications should prefer the UDUNITS-2 API directly. Legacy functions are primarily intended to support incremental migration of existing codebases.

    Functions

    Function Description
    utConvert() Computes slope and intercept needed to convert numeric values between compatible units
    utCopy() Creates a copy of a legacy unit structure
    utFree() Releases resources associated with a legacy unit structure
    utIsTime() Determines whether a parsed unit represents a time quantity
    utPrint() Formats a legacy unit structure as a string representation
    utScan() Parses a unit specification string into a legacy unit structure

    1. Input unit expressions

      Operations begin with unit expressions represented as strings, typically originating from application input, configuration data, or external sources.

      Examples:

      • "m"
      • "km"
      • "kg·m^2/s^2"

      These textual representations describe the dimensional structure of the unit.

    2. Parse unit specification using utScan()

      The unit expression is interpreted by:

      • utScan()

      utScan() parses the string and produces a legacy unit structure that internally corresponds to a UDUNITS-2 ut_unit.

      This structure encodes the dimensional relationships of the unit independent of its textual representation.

    3. Legacy unit structure available

      The resulting structure provides a normalized internal representation of the unit, capturing dimensional composition and scaling relationships.

      This representation enables compatibility evaluation and conversion operations.

    4. Inspect or duplicate unit structures

      Applications may evaluate characteristics of the parsed unit or create copies for reuse in subsequent operations.

      • utIsTime() determines whether the unit represents a temporal quantity
      • utCopy() duplicates an existing unit structure

      These operations support validation, reuse, and preservation of original unit definitions.

    5. Determine conversion relationship using utConvert()

      Compatibility between units is evaluated implicitly when conversion is requested.

      • utConvert()

      utConvert() computes linear transformation coefficients describing the relationship between two compatible unit structures.

      The transformation is expressed as:

       y = slope * x + intercept

      These coefficients correspond internally to relationships represented by cv_converter objects in UDUNITS-2 Unit Conversion API.

    6. Conversion coefficients available

      The computed coefficients define the mathematical transformation required to convert numeric values between compatible units.

      These values may be applied directly within application logic.

    7. Format unit representation using utPrint()

      Legacy unit structures may be converted into standardized string representations.

      • utPrint()

      Formatting behavior corresponds conceptually to ut_format() in the Unit Parsing & Formatting API.

      Formatted output may be used for display, logging, or serialization.

    8. Output formatted unit string

      The resulting string representation reflects the normalized dimensional structure of the unit.

      This output may differ from the original input expression while preserving dimensional equivalence.

    9. Free legacy unit structure using utFree()

      Legacy unit structures should be released when no longer needed.

      • utFree()

      Explicit resource cleanup ensures proper lifecycle management consistent with memory handling expectations of legacy applications.

    Migration To UDUNITS-2 API

    The UDUNITS-1 legacy interface maps directly to the UDUNITS-2 object model, which is based on ut_system, ut_unit, and cv_converter objects.

    Applications using the UDUNITS-1 interface can typically migrate incrementally by replacing legacy functions with their UDUNITS-2 equivalents.

    UDUNITS-1 Concept UDUNITS-2 Equivalent
    legacy unit structure ut_unit
    implicit system context ut_system
    conversion slope/intercept cv_converter
    utConvert() ut_get_converter()
    utCopy() ut_clone()
    utFree() ut_free()
    utPrint() ut_format()
    utScan() ut_parse()

    Migrating to the UDUNITS-2 API provides:

    • explicit ut_system management
    • improved dimensional consistency validation through explicit compatibility checks in the Unit Conversion API and dimensional operations in the Unit Arithmetic API
    • extended unit arithmetic capabilities
    • improved error handling configuration
    • clearer object lifecycle semantics
