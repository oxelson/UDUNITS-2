    Configure Script Options

    UDUNITS-2 Documentation Configure Script Options

    Before compiling UDUNITS-2, the build system must be configured for the local environment. This step detects compilers, system libraries, and platform features, and allows customization of installation paths and optional features.

    Configuration is performed using the configure script included in the source distribution.

    ./configure

    In most environments, running configure with no options is sufficient.

    Common Configuration Workflow

    The typical build workflow is:

    ./configure
make
make install

    Additional options can be passed to configure to control installation paths, build features, and platform settings.

    Configuration Categories

    The configure script supports several categories of options:

    Category Purpose
    Installation directories Control where files are installed
    Program naming Modify installed executable names
    System types Control platform and cross-compilation settings
    Optional features Enable or disable build features
    Optional packages Specify external dependencies
    Environment variables Override compiler and linker settings

    The sections below describe the most commonly used options.

    Installation Directories

    These options control where UDUNITS-2 files are installed.

    The most common option is --prefix, which sets the root installation directory.

    ./configure --prefix=/opt/udunits

    Default installation prefix:

    /usr/local

    Important directory options include:

    Option Description Default
    --prefix=PREFIX Root installation directory /usr/local
    --exec-prefix=EPREFIX Architecture-dependent files PREFIX
    --bindir=DIR User executables EPREFIX/bin
    --libdir=DIR Libraries EPREFIX/lib
    --includedir=DIR C header files PREFIX/include
    --datadir=DIR Architecture-independent data PREFIX/share
    --mandir=DIR Manual pages PREFIX/share/man
    --docdir=DIR Documentation PREFIX/share/doc/udunits

    Example:

    ./configure   --prefix=/opt/UDUNITS-2   --libdir=/opt/udunits/lib64

    Program Name Customization

    The configure script provides options that allow the names of installed executables to be modified during installation.

    These options affect only executable programs such as the udunits2 command. They do not affect libraries or header files.

    Option Description
    --program-prefix=PREFIX Adds a prefix to installed program names
    --program-suffix=SUFFIX Adds a suffix to installed program names
    --program-transform-name=PROGRAM Applies a sed transformation to program names

    Example:

    ./configure --program-prefix=test-

    Installs:

    test-udunits2

    These options are rarely needed for normal installations but may be useful when installing multiple versions of a program.

    System Types

    The configure script can optionally specify the systems involved in the build process.

    Option Description
    --build=BUILD System performing the compilation
    --host=HOST System where the compiled program will run

    In most cases these options are unnecessary because configure automatically detects the build environment.

    They are primarily used for cross-compilation.

    Example:

    ./configure --host=aarch64-linux-gnu

    This tells the build system that the program will run on an ARM system even though it is being compiled on another platform.

    Optional Build Features

    UDUNITS-2 supports several optional build features that can be enabled or disabled.

    Option Description
    --enable-shared Build shared libraries
    --enable-static Build static libraries
    --enable-debug Enable debugging support
    --enable-coverage Enable code coverage instrumentation
    --disable-udunits-1 Disable legacy UDUNITS-1 API compatibility

    Example:

    ./configure --enable-debug

    Optional Packages

    The configure script can optionally use external packages when available.

    Option Description
    --with-PACKAGE Use a specified package
    --without-PACKAGE Disable a package

    These options are primarily used by system package maintainers.

    Environment Variables

    Environment variables can be used to override the compiler and linker used during the build.

    Common variables include:

    Variable Purpose
    CC C compiler
    CFLAGS C compiler flags
    CPPFLAGS Preprocessor flags (include directories)
    LDFLAGS Linker flags
    LIBS Additional libraries

    Example:

    CC=gcc CFLAGS="-O2" ./configure

    Example specifying custom library paths:

    CPPFLAGS="-I/opt/include" LDFLAGS="-L/opt/lib" ./configure

    Configure Examples

    The following examples demonstrate common ways to configure a UDUNITS-2 build.

    Standard Installation

    Uses default settings and installs to /usr/local.

    ./configure
make
make install

    Custom Installation Prefix

    Install UDUNITS-2 into a custom directory.

    ./configure --prefix=/opt/udunits
make
make install

    Resulting directories:

    /opt/udunits/bin
/opt/udunits/lib
/opt/udunits/include

    Debug Build

    Enable debugging support for development or troubleshooting.

    ./configure --enable-debug
make

    This typically adds debugging symbols and may disable certain optimizations.

    Static Library Build

    Build only static libraries.

    ./configure --disable-shared --enable-static
make

    This is useful for static linking environments or embedded systems.

    Shared Library Build Only

    Build only shared libraries.

    ./configure --enable-shared --disable-static
make

    Custom Compiler

    Specify a particular compiler and optimization flags.

    CC=clang CFLAGS="-O3" ./configure
make

    Custom Library and Header Paths

    Specify non-standard include and library directories.

    CPPFLAGS="-I/opt/local/include" LDFLAGS="-L/opt/local/lib" ./configure

    Cross Compilation Example

    Build UDUNITS-2 on one architecture for another platform.

    Example: building on x86_64 Linux for ARM.

    ./configure --host=aarch64-linux-gnu
make

    The build system will attempt to use the appropriate cross-compilation toolchain.

    Viewing All Configuration Options

    To see the full list of supported configuration options:

    ./configure --help
