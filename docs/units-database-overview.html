    Units Database Overview

    UDUNITS-2 Documentation Units Database Overview

    The UDUNITS-2 database provides the authoritative definitions of measurement units used by the UDUNITS-2 library to parse unit expressions, verify dimensional compatibility, and compute conversion factors. It serves as the foundational knowledge base that enables the library to interpret textual unit specifications and transform them into structured representations that support arithmetic and conversion operations.

    The database is distributed primarily as XML files (for example udunits2.xml) that define the relationships among units, prefixes, dimensions, and constants. These definitions are loaded at runtime to construct an internal representation of a unit system (ut_system) used by the UDUNITS-2 C API.

    The database fulfills several primary roles:

    Provide Canonical Unit Definitions

    The database establishes a consistent set of definitions for measurement units across scientific and technical domains. Each unit is defined in terms of underlying physical dimensions and relationships to other units.

    Examples of definitions include:

    • SI base units such as meter, kilogram, second, kelvin
    • derived units such as newton, joule, watt
    • scaled units such as kilometer or millisecond
    • offset units such as degree Celsius
    • dimensionless units such as percent or radian

    By defining units in terms of physical relationships rather than fixed conversion tables, the database ensures that conversions remain consistent and extensible.

    Enable Parsing of Unit Expressions

    UDUNITS-2 allows users to specify units as text expressions (e.g., "kg*m/s^2" or "W/m^2"). The database provides the vocabulary and structural relationships needed for the parser to interpret these expressions.

    During parsing:

    1. Unit names and symbols are resolved using definitions from the XML database.
    2. Prefixes are applied according to prefix rules.
    3. The mathematical structure of the expression is interpreted.
    4. A corresponding ut_unit object is created.

    Because the database encodes both naming conventions and dimensional structure, it supports flexible yet unambiguous expression syntax.

    Support Dimensional Consistency and Conversion

    The database encodes dimensional relationships that allow UDUNITS-2 to determine whether two units are compatible and, if so, how to convert between them.

    For example, the database definition of the newton as:

    kg*m/s^2

    allows the library to automatically determine compatibility between:

    N
kgm/s^2
lbft/s^2

    Conversion factors are derived algorithmically from these relationships rather than stored explicitly for every possible unit pair.

    This approach ensures:

    • mathematically consistent conversions
    • reduced redundancy in stored definitions
    • extensibility for new unit types

    Provide a Configurable and Extensible Unit System

    The XML-based design allows the database to be extended or customized without modifying library source code.

    Users may:

    • add domain-specific units
    • define alternative symbols or aliases
    • override or augment default definitions
    • construct restricted unit systems for specific applications

    The modular structure of the XML database supports reuse and incremental extension through file imports.

    Serve as the Interface Between Textual Units and Internal Objects

    At runtime, the database is loaded into an internal unit system object (ut_system). This object acts as the central registry of known units used throughout the UDUNITS-2 API.

    Key relationships:

    Database Component Internal Representation
    unit definitions ut_unit objects
    unit system ut_system object
    conversion relationships cv_converter objects
    prefixes scale factors applied during parsing

    Through this mapping, the database bridges human-readable unit expressions and the internal mathematical structures required for conversion and compatibility analysis.

    1.6 Relationship to the UDUNITS-2 Architecture

    Within the overall UDUNITS-2 architecture, the database functions as the semantic layer supporting higher-level operations:

    XML database → parser → unit objects → converters

    The database does not perform conversions directly; rather, it provides the definitions that allow the library to derive conversions dynamically.

    This separation of concerns allows:

    • stable and inspectable unit definitions
    • consistent behavior across applications
    • maintainable extension of supported units
