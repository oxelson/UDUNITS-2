    Troubleshooting Installations

    UDUNITS-2 Documentation Troubleshooting Installations

    Troubleshooting Package Manager Installations

    Most installation issues when installing UDUNITS-2 using system package managers fall into a few common categories.

    Examples of package managers include apt, dnf/yum, Homebrew, and MacPorts.

    Before using the troubleshooting tables below, run the checks in the UDUNITS-2 Installation Health Check to verify that the UDUNITS-2 executable, headers, libraries, and units database are correctly installed.

    Development Headers Missing

    Symptom Cause Diagnosis Fix
    udunits2.h: No such file or directory Development headers not installed Check installed packages Install development package (libudunits2-dev, udunits2-devel, etc.)
    Python/R packages fail detecting UDUNITS-2 Runtime package installed but headers missing Inspect build logs for udunits2.h errors Install the UDUNITS-2 development package

    Package Metadata Issues

    Symptom Cause Diagnosis Fix
    Package 'udunits2' not found from pkg-config pkg-config metadata missing Run pkg-config --list-all | grep udunits Install development package containing udunits2.pc

    Library Linking Issues

    Symptom Cause Diagnosis Fix
    ld: cannot find -ludunits2 Linker cannot find the UDUNITS-2 library Run pkg-config --libs udunits2 Install development package or add correct -L path
    undefined reference to ut_* functions Program not linking against UDUNITS-2 Inspect compile command Add -ludunits2 to linker flags

    Runtime Issues

    Symptom Cause Diagnosis Fix
    Runtime error loading libudunits2.so / .dylib Dynamic linker cannot find library Run ldd program or otool -L program Update linker cache (ldconfig) or adjust library paths
    Couldn't open units database XML database path overridden or missing Check echo $UDUNITS2_XML_PATH Unset variable or reinstall package

    Environment Conflicts

    Symptom Cause Diagnosis Fix
    Software builds but behaves unexpectedly Multiple UDUNITS-2 installations present Run which udunits2 and inspect linked libraries Remove duplicate installs or ensure consistent include/library paths

    Troubleshooting Source Builds

    When building UDUNITS-2 from source, installation issues typically occur during one of the following stages:

    Dependencies → Build Configuration → Compilation and Linking → Runtime → Environment Conflicts → Platform-Specific Issues

    Before using the troubleshooting tables below, run the checks in the UDUNITS-2 Installation Health Check to verify that the UDUNITS-2 executable, headers, libraries, and units database are correctly installed.

    Dependency Issues

    These errors occur when required libraries cannot be found during the configuration stage.

    Symptom Cause Diagnosis Fix
    configure: cannot find EXPAT function XML_StopParser Expat library missing or not detected Inspect config.log Install the Expat development package
    expat.h: No such file or directory Expat headers not installed Search include directories Install Expat development headers
    ld: cannot find -lexpat Linker cannot locate Expat library Check library paths Install Expat library or add -L/path/to/expat/lib

    Build Configuration Issues

    These problems occur during the configure or CMake stage when paths or environment variables are not set correctly.

    Symptom Cause Diagnosis Fix
    Configure fails to detect dependencies Environment variables not configured correctly Inspect config.log Set CPPFLAGS and LDFLAGS appropriately
    udunits2.h: No such file or directory Headers installed in non-standard location Locate installed headers Add -I/path/to/include

    Compilation And Linking Issues

    These errors occur when compiling or linking applications against UDUNITS.

    Symptom Cause Diagnosis Fix
    ld: cannot find -ludunits2 Library path not visible to linker Inspect library install location Add -L/path/to/lib
    undefined reference to ut_* functions Program not linking against UDUNITS-2 Inspect compile command Add -ludunits2

    Runtime Issues

    These problems appear after successful compilation when the program is executed.

    Symptom Cause Diagnosis Fix
    Runtime error loading libudunits2.so Library path not registered with dynamic linker Run ldd program Run ldconfig or update LD_LIBRARY_PATH
    Couldn't open units database XML units database not found Search installation directory for udunits2.xml Set UDUNITS2_XML_PATH

    Environment and Installation Conflicts

    These issues occur when multiple versions of UDUNITS-2 exist on the system.

    Symptom Cause Diagnosis Fix
    Program links against wrong UDUNITS-2 version Multiple installations present Inspect linked libraries with ldd Remove duplicate installations or adjust include/library paths

    Windows-Specific Issues

    These problems occur when building UDUNITS-2 using Visual Studio or Windows toolchains.

    Symptom Cause Diagnosis Fix
    udunits2.h not found in Visual Studio Include path not configured Check project include directories Add UDUNITS-2 include directory
    udunits2.lib not found during linking Import library not configured Inspect linker settings Add library directory and dependency
    libudunits2.dll not found at runtime DLL not in system search path Run program from command prompt Add directory to PATH or copy DLL next to executable

    Troubleshooting Conda Installations

    Most Conda installation issues fall into a small number of common categories.

    Before using the troubleshooting tables below, run the checks in the UDUNITS-2 Installation Health Check to verify that the UDUNITS-2 executable, headers, libraries, and units database are correctly installed.

    Package Missing From Environment

    Symptom Cause Diagnosis Fix
    udunits2.h: No such file or directory UDUNITS-2 not installed in environment Run conda list | grep udunits Install with conda install udunits2

    Environment Activation Issues

    Symptom Cause Diagnosis Fix
    Python package build fails detecting UDUNITS-2 Environment not activated Run conda info --envs Activate environment with conda activate <env>

    Library Loading Issues

    Symptom Cause Diagnosis Fix
    DLL load failed or Library not loaded: libudunits2 Environment library paths inactive Inspect linked libraries Ensure environment is activated before running program

    Language Binding Issues

    Symptom Cause Diagnosis Fix
    R package units fails to install UDUNITS-2 headers missing in environment Inspect installation logs Install udunits2 inside environment

    Path Conflicts

    Symptom Cause Diagnosis Fix
    Program links against system UDUNITS-2 System paths appear before Conda paths Inspect libraries with ldd or otool -L Ensure Conda environment appears first in PATH

    IDE Environment Issues

    Symptom Cause Diagnosis Fix
    Package works in terminal but not IDE IDE not using Conda environment Check interpreter settings Configure IDE to use correct environment

    Rare Runtime Issues

    Symptom Cause Diagnosis Fix
    Runtime unit parsing errors XML database not found in environment Locate udunits2.xml Ensure UDUNITS2_XML_PATH is not overriding correct path
