    Conda Installation

    UDUNITS-2 Documentation Conda Installation

    This page describes how to install UDUNITS-2 using Conda-based environments (Conda, Mamba, Micromamba) for use with C, C++, Python, R, and other scientific software.

    When To Use Conda

    Conda-based installation is recommended if you:

    • Work in Python or R environments
    • Need cross-platform consistency (Linux, macOS, Windows)
    • Want to avoid system-level package managers
    • Build software in isolated or reproducible environments (CI, containers)

    Conda is often the simplest and most portable way to obtain UDUNITS-2, especially on Windows.

    Supported Platforms

    Platform Status Notes
    Linux ✅ Fully supported x86_64, aarch64
    macOS ✅ Fully supported Intel and Apple Silicon
    Windows ✅ Fully supported Native Conda packages

    Conda packages provide both runtime libraries and development headers in a single environment.

    Supported Windows Environments

    For most Windows users, Conda is the only practical and supported approach.

    Conda provides prebuilt UDUNITS-2 binaries for Windows, manages runtime paths automatically, and automatic DLL placement and PATH configuration.

    Environment Status Notes
    Anaconda / Miniconda ✅ Supported Most common choice
    Mamba / Micromamba ✅ Supported Faster dependency resolution
    Conda-forge channel ✅ Required Do not mix with defaults

    Conda Channels

    UDUNITS-2 is distributed via the conda-forge channel, which provides up-to-date, community-maintained scientific packages.

    Note: Always prioritize conda-forge for UDUNITS-2. Mixing channels can lead to ABI incompatibilities.

    Creating A Conda Environment

    Create a dedicated environment to avoid dependency conflicts:

    conda create -n udunits-env
conda activate udunits-env

    Using mamba (recommended for speed):

    mamba create -n udunits-env
mamba activate udunits-env

    Installing UDUNITS-2

    Install UDUNITS-2 from conda-forge:

    conda install -c conda-forge udunits2

    This installs UDUNITS-2 and the required dependencies (e.g., Expat). No separate “development package” is required.

    Files Installed

    Within an active Conda environment, the following UDUNITS-2 files are located at:

    Component Typical Location
    UDUNITS-2 Binary $CONDA_PREFIX/bin/udunits2
    %CONDA_PREFIX%\Library\bin\udunits2.exe (Windows)
    Header File $CONDA_PREFIX/include/udunits2.h
    %CONDA_PREFIX%\Library\include\udunits2.h (Windows)
    Shared Library $CONDA_PREFIX/lib/libudunits2.*
    %CONDA_PREFIX%\Library\bin\libudunits2.dll (windows
    XML Units Database $CONDA_PREFIX/share/udunits/udunits2.xml
    %CONDA_PREFIX%\Library\share\udunits\udunits2.xml (Windows)

    In Windows environments, Conda automatically ensures that the correct DLLs are discoverable at runtime.

    Environment Variables

    Conda environments automatically configure:

    • PATH
    • Library search paths (rpath / platform-specific mechanisms)

    In most cases, no manual environment variables are required.

    Warning: Avoid manually setting LD_LIBRARY_PATH or UDUNITS2_XML_PATH inside Conda environments unless you fully understand the implications.

    Verifying Installation

    After installing UDUNITS-2, verify that the installation is working before building dependent software.

    A correct installation requires the following components to be discoverable:

    • the udunits2 executable
    • the development headers (udunits2.h)
    • the UDUNITS-2 library (libudunits2)
    • the units database (udunits2.xml)

    The Installation Health Check guide provides commands and scripts that confirm these components are installed and accessible. It can quickly detect common issues such as missing headers, missing libraries, incorrect include or library paths, or a misconfigured units database.

    Troubleshooting

    Most Conda installation issues fall into a small number of common categories.

    Before using the troubleshooting tables below, run the checks in the UDUNITS-2 Installation Health Check to verify that the UDUNITS-2 executable, headers, libraries, and units database are correctly installed.

    Package Missing From Environment

    Symptom Cause Diagnosis Fix
    udunits2.h: No such file or directory UDUNITS-2 not installed in environment Run conda list | grep udunits Install with conda install udunits2

    Environment Activation Issues

    Symptom Cause Diagnosis Fix
    Python package build fails detecting UDUNITS-2 Environment not activated Run conda info --envs Activate environment with conda activate <env>

    Library Loading Issues

    Symptom Cause Diagnosis Fix
    DLL load failed or Library not loaded: libudunits2 Environment library paths inactive Inspect linked libraries Ensure environment is activated before running program

    Language Binding Issues

    Symptom Cause Diagnosis Fix
    R package units fails to install UDUNITS-2 headers missing in environment Inspect installation logs Install udunits2 inside environment

    Path Conflicts

    Symptom Cause Diagnosis Fix
    Program links against system UDUNITS-2 System paths appear before Conda paths Inspect libraries with ldd or otool -L Ensure Conda environment appears first in PATH

    IDE Environment Issues

    Symptom Cause Diagnosis Fix
    Package works in terminal but not IDE IDE not using Conda environment Check interpreter settings Configure IDE to use correct environment

    Rare Runtime Issues

    Symptom Cause Diagnosis Fix
    Runtime unit parsing errors XML database not found in environment Locate udunits2.xml Ensure UDUNITS2_XML_PATH is not overriding correct path

    Best Practices

    • Always activate the Conda environment before building or running software
    • Use conda-forge exclusively for scientific stacks
    • Keep UDUNITS-2 inside the same environment as dependent software
    • Avoid mixing Conda libraries with system package-manager installs
    • Verify installation immediately after install is complete

    NetCDF + UDUNITS-2 Integration

    Conda environments install NetCDF and UDUNITS-2 together, ensuring compatible versions of the libraries and their dependencies. This approach is commonly recommended for Python workflows, R environments, Windows users, and reproducible scientific environments, because Conda manages the full dependency stack automatically.

    For example, you can create an environment and install the libraries with:

    conda create -n netcdf-env
conda activate netcdf-env
conda install -c conda-forge netcdf udunits2

    For R-based workflows, Conda can also install the required R packages alongside the system libraries:

    conda create -n r-netcdf
conda activate r-netcdf
conda install -c conda-forge r-base r-udunits2 r-ncdf4 udunits2 netcdf

    After installation, you can verify that NetCDF was built with UDUNITS-2 support using:

    nc-config --has-udunits

    A yes response indicates that the integration is available. For best results, keep all NetCDF-related packages within the same Conda environment, prefer the conda-forge channel, and avoid mixing Conda libraries with system-installed versions.

    Language Bindings Notes

    C/C++ Projects

    1. Compilation examples for linux and macOS environments

       gcc example.c \
   -I$CONDA_PREFIX/include \
   -L$CONDA_PREFIX/lib \
   -ludunits2

      If you are using pkg-config:

      Conda-forge provides pkg-config metadata:

       pkg-config --cflags udunits2
 pkg-config --libs udunits2

      Ensure pkg-config is installed in the environment:

       conda install -c conda-forge pkg-config

    2. Windows Example (MinGW / Conda Toolchain):

       gcc example.c ^
   -I%CONDA_PREFIX%\Library\include ^
   -L%CONDA_PREFIX%\Library\lib ^
   -ludunits2
      Warning: Mixing Conda-provided libraries with MSVC toolchains is not recommended unless you fully control ABI compatibility.

    Python Projects

    Many Python packages that rely on UDUNITS-2 (directly or indirectly) work best in Conda environments:

    conda install -c conda-forge udunits2 python

    This avoids system-level header and linker issues commonly seen with pip builds.

    R Projects

    R packages such as udunits2 from CRAN link cleanly against Conda-provided UDUNITS-2:

    conda install -c conda-forge r-base udunits2
