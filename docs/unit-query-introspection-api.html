    Unit Query & Introspection API

    UDUNITS-2 Documentation Unit Query & Introspection API

    Functions for retrieving unit properties, metadata, and performing comparisons.

    Some returned values are system-owned and must not be freed by the caller.

    Functions

    Function Description
    ut_compare() Compares two ut_unit objects for equivalence or ordering relationship
    ut_get_name() Retrieves the canonical name associated with a ut_unit
    ut_get_path_xml() Returns the filesystem path of the XML database used to initialize a ut_system
    ut_get_symbol() Retrieves the standard symbol associated with a ut_unit
    ut_get_unit_by_name() Returns a ut_unit from a ut_system based on a canonical unit name
    ut_get_unit_by_symbol() Returns a ut_unit from a ut_system based on a standard unit symbol
    ut_is_dimensionless() Indicates whether a ut_unit represents a dimensionless quantity

    Usage Patterns

    The Unit Query & Introspection API operates as a progression from identifier or existing ut_unit → property inspection → identifier lookup or comparison → formatted output or system metadata, enabling structured discovery, validation, and presentation of unit characteristics.

    1. Input unit expression

      Operations begin with a unit expressed as a string.

      Example:

      "kg·m^2/s^2"

      This string representation typically originates from user input, configuration data, or external files.

    2. Parse string and construct ut_unit object

      The unit expression is converted into the UDUNITS-2 internal representation.

      • ut_parse() converts the string into a ut_unit within a ut_system

      This step produces the ut_unit object required for introspection, comparison, and formatting operations.

    3. ut_unit representation available

      A ut_unit instance provides a structured representation of the unit’s dimensional composition and scaling relationships.

      The ut_unit may represent a base unit, derived unit, or compound expression and serves as the primary object used throughout Query and Introspection workflows.

    4. Property introspection

      Descriptive attributes associated with the ut_unit can be retrieved.

      • ut_get_name() returns the canonical name associated with the ut_unit
      • ut_get_symbol() returns the standard symbol associated with the ut_unit

      These functions support labeling, diagnostics, and display logic.

    5. Lookup by identifier

      A ut_system can resolve identifiers such as canonical names or symbols into corresponding ut_unit objects.

      • ut_get_unit_by_name() returns a ut_unit from a ut_system using a canonical name
      • ut_get_unit_by_symbol() returns a ut_unit from a ut_system using a standard symbol

      This supports dynamic resolution of unit identifiers to structured objects.

    6. Validate or compare units

      Structural or relational properties of ut_unit objects can be evaluated.

    • ut_compare() compares two ut_unit objects for equivalence or ordering relationships

    • ut_is_dimensionless() determines whether a ut_unit has no physical dimensions

      These operations support validation logic and compatibility checks.

    1. Format or display unit metadata

      A ut_unit can be converted into a readable string representation.

      • ut_format() formats a ut_unit into a textual representation

      This is commonly used for logging, reporting, or UI display.

    2. Access system-level metadata

      Metadata describing the originating ut_system can be retrieved.

      • ut_get_path_xml() returns the filesystem path of the XML database used to initialize the ut_system

      This supports diagnostics and configuration transparency.

    3. Output metadata or comparison results

      The application receives structured information derived from ut_unit and ut_system objects, such as:

      • canonical name or symbol
      • formatted unit expression
      • comparison results between ut_unit objects
      • dimensional characteristics
      • unit system configuration details
