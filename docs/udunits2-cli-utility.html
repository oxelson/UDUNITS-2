    udunits2 CLI Utility

    UDUNITS-2 Documentation udunits2 CLI Utility

    Overview

    The UDUNITS-2 package provides a command-line utility called udunits2 for converting numeric values between compatible physical units using the UDUNITS-2 unit database.

    The tool supports unit parsing, dimensional analysis, and numeric conversion across a wide range of scientific and engineering units.

    The udunits2 CLI utility uses the same underlying parsing and conversion functionality exposed through the C API.

    Note: Earlier documentation may refer to the CLI utility as udunits2prog. The installed executable is named udunits2.

    Basic Syntax

    udunits2 [options]

    For example:

    udunits2 -H "<value> <unit>" -W "<target-unit>"

    Command Options

    Option Description
    -H input quantity (value + units)
    -W desired output units
    -L specify XML unit database file
    -h show help
    -V show version

    Usage

    udunits2 can be used in both non-interactive and interactive modes.

    Non-Interactive Use

    In non-interactive mode, the input quantity and desired output units are provided directly on the command line. This approach is well suited for scripts, automation, and quick one-line conversions.

    For example, convert meters per second to kilometers per hour:

    udunits2 -H "10 m/s" -W "km/h"

    Output:

    10 m/s = 36 km/h

    Because all inputs are supplied as command-line arguments, the program performs the conversion immediately and exits.

    Interactive Use

    When run without conversion arguments, udunits2 enters an interactive prompt that allows repeated conversions within a single session.

    For example:

    udunits2

    Example session:

    You have: 10 m/s
You want: km/h
    10 m/s = 36 km/h

    Interactive mode is useful for exploratory work, checking unit relationships, or performing multiple conversions without repeatedly invoking the command.

    Exit the interactive session using:

    Ctrl+D

    Unit Expression Syntax

    udunits2 accepts unit expressions composed of standard mathematical operators, numeric values, and recognized unit symbols from the UDUNITS database. These expressions define the dimensional structure of quantities and are used to evaluate compatibility and perform conversions.

    The following table summarizes common syntax patterns supported by the UDUNITS-2 parser.

    Syntax Type Example Description
    multiplication kg*m, N*m combine units multiplicatively
    division m/s, kg/m^3 express ratios of units
    powers m^2, s^-1, m^0.5 raise units to integer or fractional exponents
    scientific notation 1e3 m, 9.81 m/s^2 express numeric values using exponential notation
    derived units Pa, J, W, Hz predefined unit symbols in the UDUNITS database
    time units hours, minutes, days, seconds supported representations of time

    Relationship To C API

    The CLI utility provides a user-facing interface to the same functionality available through the UDUNITS-2 C API.

    CLI capability C API equivalent
    parse unit expressions ut_parse()
    check compatibility ut_are_convertible()
    perform conversions ut_get_converter()
    format unit expressions ut_format()
