    UDUNITS-2 Installation Overview

    UDUNITS-2 Documentation UDUNITS-2 Installation Overview

    This page is the starting point for installing UDUNITS-2. It helps you choose the correct installation method and links to detailed, method-specific guides.

    Choose Your Installation Method

    UDUNITS-2 is commonly used on its own, but most often appears as a dependency of NetCDF, R, Python, and other scientific software.

    Choosing the right installation method early avoids the majority of build and runtime issues.

    Method Best For Goals Platforms Notes
    System Package Manager Most users System-wide installation, easy maintenance, automatic updates Linux, macOS Recommended when available. Integrates with the operating system and handles dependencies automatically
    Conda / Conda-Forge Cross-platform environments and scientific workflows Cross-platform consistency, reproducible environments, environment isolation Linux, macOS, Windows Good option when the same environment must run across multiple platforms
    Build from Source Developers or advanced users Full control over build configuration, custom installation paths, debugging, or contributing to development Linux, macOS Primary method when package managers are unavailable or customization is required
    Build from Source (Windows) Windows users needing the native C library Cross-platform development or integration with Windows toolchains Windows Secondary option on Windows when Conda packages are not suitable. Requires a C toolchain and dependency setup
    Tip: Use one installation method per environment. Mixing methods is the most common cause of failures.

    Installation Guides

    Follow exactly one of the guides below that matches your chosen method:

    Package Manager Installation

    Best for system-level installations on Linux and macOS.

    • Uses OS-provided packages
    • Requires separate development packages for building software
    • Integrates naturally with system NetCDF installs

    Typical users: system administrators, OS-level NetCDF users

    ➡️ See: Package Manager Installation

    Build from Source

    Best for controlled environments where you need full control over versions, install locations. and toolchains.

    • Custom installation prefixes (e.g., /opt, $HOME/local)
    • Common on HPC systems and CI pipelines
    • Requires manual dependency management

    Typical users: HPC users, CI maintainers, developers

    ➡️ See: Building from Source

    Conda Installation

    Best for cross-platform workflows, language-focused ecosystems, and reproducible environments.

    • Works on Linux, macOS, and Windows
    • Includes headers and libraries automatically
    • Strongly recommended for Python and R

    Typical users: data scientists, Windows users, mixed-language projects

    ➡️ See: Conda Installation

    Verify Your Installation

    After installing UDUNITS-2, verify that the installation is working before building dependent software.

    A correct installation requires the following components to be discoverable:

    • the udunits2 executable
    • the development headers (udunits2.h)
    • the UDUNITS-2 library (libudunits2)
    • the units database (udunits2.xml)

    The Installation Health Check guide provides commands and scripts that confirm these components are installed and accessible. It can quickly detect common issues such as missing headers, missing libraries, incorrect include or library paths, or a misconfigured units database.

    ➡️ See: Installation Health Check

    Diagnosing Installation Problems

    If software cannot detect UDUNITS, or build/runtime errors occur, consult the Troubleshooting Installations guide.

    The troubleshooting guide provides platform-specific diagnostic tables for package manager installs, source builds, Conda environments, and Windows systems. Each table lists common symptoms, likely causes, diagnostic commands, and recommended fixes.

    Before consulting these tables, reference the Installation Health Check to verify the state of your installation.

    ➡️ See: Troubleshooting Installations
